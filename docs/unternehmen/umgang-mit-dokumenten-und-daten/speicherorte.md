## Dateinamen und Versionierung

### Grundsätze

- Jedes Dokument, das in Versionen abgespeichert werden wird (etwa ein Buchprojekt.indd), erhält eine Versionsnummer.
- Jedes Dokument wird irgendwann weiterentwickelt. Es gibt daher keine fertigen Dokumente.
- Bezeichnungen wie def, definitiv, neu, neu-2, … sind absolut unerwünscht. Sie folgen keinem nachvollziehbaren Schema und sind nur durch die eigene Erinnerung aussagekräftig.

### Regeln

- Die Versionierung ist eine fortlaufende Nummer 1, 2, 3, …
- Die Versionsnummer wird dem Dokumentnamen angehängt: `Buchprojekt v13`
- Exportierte Daten (etwa ein PDF aus einer INDD) heisst wie die INDD Datei, das Datum wird ans Ende angehängt: `Buchprojekt v13 220428`

## Passwörter

### Regeln für den Umgang mit Passwörter

1. Verwende ausnahmslos für jedes Login ein eigenes Passwort
2. Speichere alle Passwörter in der Passwort-App. Immer.
3. Speichere Passwörter nie in Textdateien oder sonst wo
4. Erzeuge alle Passwörter mit der Passwort-App, nie händisch. Ausnahme: Das Passwort für die Passwort-App.
5. Benutze die Passwortlängen, die im vertraulichen Teil des Unternehmenshandbuchs dokumentiert sind
6. Versende Passwörter nie offen per Mail. Nutze stattdessen die Passwort teilen Funktion aus 1Password.
7. Wo möglich, aktiviere 2-Faktor-Authentifizierung (2FA)
8. Auf Deinem eigenen Computer darfst Du Passwörter im Browser speichern. Auf fremden Rechnern nie.
